1. Executive Summary & Educational Objectives

CodeLens is a state-of-the-art, web-based Integrated Development Environment (IDE) designed to bridge the gap between traditional online code runners and advanced AI coding assistants. Unlike simple web run-boxes that merely echo raw terminal output, CodeLens provides a deeply structured, highly visual, and educational coding experience. It empowers software engineering students, educators, and developers to write, compile, execute, debug, and understand code across five major programming languages (Python, C++, C, Java, and JavaScript) inside a secure, sandboxed cloud runtime.

Why Study This Project for Project-Based Learning (PBL)?

Building an online IDE from scratch touches every critical layer of modern full-stack systems engineering:

  • Frontend Virtualization & AMD Module Loading: Integrating industry-standard editor engines (monaco-editor used by VS Code) without heavy bundlers.
  • Subprocess & OS Execution Management: Safely spawning child processes (`child_process.spawn`) to invoke native system compilers (`gcc`, `g++`, `javac`) and interpreters while enforcing execution timeouts.
  • Complex Regular Expression Error Parsing: Translating cryptic compiler stack traces into human-readable line numbers, column numbers, and actionable educational suggestions.
  • Large Language Model (LLM) Systems Engineering: Designing strict system prompts that force LLMs (`llama-3.3-70b-versatile` via Groq) to output reliable, deeply structured JSON UI payloads.
  • Containerization & Cloud Infrastructure: Using Docker to encapsulate multi-language toolchains (`OpenJDK 17`, `GCC`, `Python 3`, `Node.js`) into reproducible, cloud-agnostic deployment units.

3.2 Virtual File System & Tab Buffer Management

To support instant, zero-lag tab switching, each open file maintains its own isolated monaco.editor.ITextModel inside the monacoModels map. When a user switches tabs, CodeLens invokes monacoEditor.setModel(monacoModels[file.id]) along with restoring exact scroll view states (monacoViewStates[id]).

4. Backend Engineering: Sandboxed Multi-Language Execution Engine

The execution engine (server/routes/run.js) is the core technical powerhouse of CodeLens. When a request arrives at POST /api/run, the server executes a strict, atomic 5-step lifecycle:

  1. Temporary Sandbox Creation: Creates an atomic, guaranteed-unique directory using fs.promises.mkdtemp(path.join(os.tmpdir(), 'codelens_run_')) to prevent file collisions between concurrent users.
  2. Code Serialization: Writes the incoming code payload to disk (`Main.java`, `main.cpp`, `main.py`, or `main.js`).
  3. Compilation Step (Java & C++ / C): Invokes native system compilers (`javac` or `g++`) with strict 10-second timeouts. If compilation fails, output is parsed immediately without running the executable.
  4. Subprocess Execution: Spawns the compiled binary or interpreter (`java -cp tmpDir Main`, `python3 main.py`, or `./a.out`), pipes user stdin into the child process, and enforces an absolute execution timeout (EXEC_TIMEOUT_MS = 10,000 ms).
  5. Resource Cleanup: Inside a strict try/finally block, recursively deletes the temporary directory (`rm(tmpDir, { recursive: true, force: true })`) to ensure zero disk leaks.

4.1 Java (`Main.java`) Execution Pipeline

// Step 1: Compile using javac
const compRes = await runCommand('javac', [codePath], { cwd: tmpDir, timeout: 10000 });

if (compRes.err || compRes.stderr) {
  // Parse and return exact compilation error structure
  const proc = processError(compRes.stderr || compRes.stdout, code, true, 'java');
  return res.json({ errorType: 'compile', message: proc.structured.message, ... });
}

// Step 2: Execute using java with classpath pointing to temporary sandbox
const runRes = await runCommand('java', ['-cp', tmpDir, 'Main'], {
  cwd: tmpDir,
  timeout: EXEC_TIMEOUT_MS,
  input: stdin || ''
});

One of the most complex engineering feats in CodeLens is translating cryptic compiler stack traces into clean, structured educational feedback.

5.1 Path Stripping & Compiler Noise Reduction

// Strip absolute Windows and Unix paths down to only the filename
sanitized = sanitized.replace(/[a-zA-Z]:\\[^\s:*?"<>|]*\\(Main\.java|main\.(?:cpp|c|py|js))/gi, '$1');
sanitized = sanitized.replace(/\/[^\s:*?"<>|]*\/(Main\.java|main\.(?:cpp|c|py|js))/gi, '$1');

// Filter out internal compiler stack noise not pointing to the user's file
lines = lines.filter(line => {
  const l = line.toLowerCase();
  if (l.includes('mingw') || l.includes('bits/stdc++.h')) return false;
  if ((l.includes('note:') || l.includes('here')) && !l.includes('main.') && !l.includes('Main.java')) return false;
  return true;
});

6. AI Integration Engine: Groq Llama-3.3-70B Versatile

CodeLens incorporates generative AI (`llama-3.3-70b-versatile`) as an automated code reviewer that returns strict, UI-parseable JSON structures.

6.1 The AI Explain Schema (`POST /api/explain`)

The backend prompts Llama-3.3 with a comprehensive 13-section JSON schema requirement. When received, the frontend renders each section into collapsible, interactive UI cards:

  • Summary & Output: High-level program purpose and predicted console output.
  • Line-by-Line Breakdown: An array of objects explaining every line of the user's code individually.
  • Variables & Functions Table: Complete symbol table with data types and functional purposes.
  • Algorithm & Complexity Analysis: Big-O Time (`timeComplexity`) and Space (`spaceComplexity`) evaluation.
  • Dry Run & Common Mistakes: Step-by-step execution trace and typical pitfalls for students.
  • Socratic Suggestions & Better Approach: Proactive code review recommending industry best practices.
  • Related Concepts & Interview Questions: Top technical interview questions tailored to the submitted code.

8. DevOps & Cloud Deployment: Dockerization, Vercel & Render

When deploying multi-language code executors to cloud infrastructure, standard Node runtimes lack JDK and C++ tooling. By containerizing CodeLens with a custom Dockerfile based on Debian 12 Bookworm (`node:20-bookworm`), we guarantee identical execution environments across local laptops and production cloud servers:

# Use official Node.js 20 on Debian Bookworm
FROM node:20-bookworm

# Install OpenJDK 17 (java + javac), Python 3, and build-essential (gcc + g++)
RUN apt-get update && apt-get install -y \
    openjdk-17-jdk \
    python3 \
    python3-pip \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY package*.json ./
RUN npm install --production
COPY . .
EXPOSE 3000
CMD ["npm", "start"]

9. Project-Based Learning (PBL) Study Guide & Student Exercises

To verify your technical understanding of the CodeLens codebase, complete these practical capstone exercises:

  1. Add Rust (`rustc`) Support: Update `Dockerfile` to include `rustc`. Add `.rs` compilation and execution handling to `server/routes/run.js` (`rustc main.rs -o main && ./main`).
  2. Implement Real-Time WebSocket Streaming: Replace the REST `POST /api/run` request with `socket.io` to stream `child.stdout` in real-time as long-running programs execute.
  3. Add Relational Database Persistence (PostgreSQL / SQLite): Replace the UUID JSON file storage in `save.js` with a relational `Users` and `Workspaces` schema using Prisma or Sequelize.
  4. Implement Sub-Container Isolation: Instead of running `g++` directly inside the main backend, spawn ephemeral Docker sub-containers (`docker run --rm --memory=128m codelens-runner`) for ultimate multi-tenant security!
